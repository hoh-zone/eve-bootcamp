# 18. 闭关锁国网关 (Border Control API)

## 💡 核心概念 (Concept)
一个极其严苛的访问控制网关。玩家进入某星区前，除了校验身份外，还自动审查其钱包中的 $SUI 余额级别或是否持有特定黑名单公会的资产。

## 🧩 解决的痛点 (Pain Points Solved)
- **内鬼渗透**：单纯的白名单容易被“临时跳槽”的间谍规避。
- **资源白嫖**：防止低价值、无产出的“肉鸡”小号随意占用核心高能矿产区。

## 🎮 详细玩法与机制 (Gameplay Mechanics)
1. **多重条件组合检查**：进入指令由 PTB 执行。第一步查询 SuiNS 域名关联性，第二步查询钱包中特定荣誉勋章（Badge）的持有量。
2. **动态拒绝逻辑**：如果该地址曾被标记为 `Treasonous`（叛徒），网关合约自动触发 `abort` 导致跳跃交易失败。
3. **资产门票**：设定“持仓进入”模式，如：钱包中有 1000 SUI 以上才允许跨过此核心枢纽。

## 🛠️ Sui 核心特性应用 (Sui Features)
- [x] SuiNS (快速身份标识检索)
- [x] PTB (在跳跃行为前置执行条件检查)

## 📐 智能合约架构规划 (Smart Contract Architecture)
### 核心 Object
- `VisaRequest`: 包含通过条件的权限证明。
- `BorderPolicy`: 指挥官实时修改的限行策略。
